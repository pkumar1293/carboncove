# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What is CarbonCove

A CBAM (Carbon Border Adjustment Mechanism) compliance SaaS for Indian exporters. Calculates EU carbon liability under EU Reg 2023/1773, compares actual vs EU-default emission intensity, and quantifies savings from real measurement data.

## Orchestrator Commands

```bash
# Full build — runs all 8 agents in phase order
python orchestrate.py

# Resume after a failure (reads .orchestrator_state.json)
python orchestrate.py --resume

# Run a specific phase only (0–4)
python orchestrate.py --phase 2

# Run a single agent only
python orchestrate.py --agent C

# Preview the execution plan without running anything
python orchestrate.py --dry-run

# Override model (default: claude-opus-4-6)
python orchestrate.py --model claude-sonnet-4-6
```

**Agent success signal**: the orchestrator looks for `AGENT-X COMPLETE` (e.g., `AGENT-C COMPLETE`) in each agent's stdout. An agent that finishes without printing this line is marked failed and retried.

## Architecture

### Monorepo Layout
```
carboncove/           ← generated by the agents into this subdirectory
├── apps/
│   ├── web/          Next.js 14 (pnpm workspaces)
│   └── api/          FastAPI Python
├── packages/shared/  TypeScript types
├── turbo.json
└── package.json      pnpm workspaces root
```

### Auth Split (Critical)
Supabase handles **auth only** (JWT tokens, email verification). All application data lives in a **local PostgreSQL** database on the VPS. FastAPI verifies Supabase JWTs by calling `SUPABASE_URL/auth/v1/user` — there is no Prisma, no Supabase DB, no Supabase RLS on application tables.

### Database (SQLAlchemy + Alembic — 11 tables)
`companies`, `users`, `installations`, `products`, `activity_data`, `emission_results`, `reports`, `leads`, `emission_factors`, `cea_grid_factors`, `eu_default_values`, `app_config`

- `app_config` — stores runtime-overridable values (`EU_ETS_PRICE_EUR`, `EUR_TO_INR_RATE`). The admin settings UI edits these directly, replacing SSH config changes.
- `users.id` matches `supabase.auth.users.id` exactly (no separate UUID).
- All migrations via Alembic (`alembic upgrade head`). Never use Prisma in `apps/api/`.

### Calculation Engine (apps/api/engine/)
Implements EU Reg 2023/1773 calculation-based method:
- `scope1.py` — fuel combustion (`E = FC × NCV × EF × oxidation_factor`) + process emissions (limestone decomp, clinker calcination, PFC for aluminium)
- `scope2.py` — grid electricity (CEA region factors) + captive power plant
- `cbam_cost.py` — `specific_embedded = total_tCO2 / production_tonnes`; compares to EU defaults per CN code; calculates savings in EUR and INR

### API Endpoints
- `POST /estimate` — **public**, rate-limited (10/min), powers the free calculator; no auth required
- `POST /calculate` — **authenticated**, saves `EmissionResult` to DB
- `POST /estimate/lead` — captures calculator leads
- `GET /admin/config`, `PUT /admin/config` — authenticated; edit `app_config` table
- `GET /files/{filename}` — authenticated file download; Nginx routes `/files/` here instead of serving uploads as static (prevents auth bypass)

### Frontend Key Patterns
- **Route groups**: `(auth)/` for login/signup, `(app)/` for the authenticated dashboard shell
- **AppContext** (`src/contexts/AppContext.tsx`) — global state: current user, company, installations, active installation; sets axios `Authorization` header on auth state change
- **sessionStorage key `cc_estimate_result`** — calculator result persisted here before step 4 lead capture to survive re-renders
- **Number formatting**: always use `formatINR()` / `formatTonne()` / `formatCO2()` from `src/lib/formatters.ts` (uses `en-IN` locale for Indian lakh system: ₹1,50,000 not ₹150,000)
- `EU_ETS_PRICE_EUR_DEFAULT` / `EUR_TO_INR_DEFAULT` in `constants.ts` are **fallbacks only** — real values come from the `app_config` DB table at runtime

### Async / Background Jobs
Celery worker (`worker.py`) backed by Redis. Tasks:
- `pdf_tasks.generate_report_task` — generates ReportLab PDF, saves to `/var/www/carboncove/uploads/reports/`
- `email_tasks.send_lead_email` — sends via Resend API

### Deployment Stack
Hostinger KVM 2 VPS (Ubuntu 24.04):
- **Nginx** reverse proxy: carboncove.in → port 3000, api.carboncove.in → port 8000
- **PM2** manages 5 processes: `carboncove-api`, `carboncove-worker`, `carboncove-web`, plus `-staging` variants on ports 8001/3001
- **Staging** on separate subdomains (`staging.carboncove.in`, `api-staging.carboncove.in`) using `carboncove_staging` DB
- CI/CD via GitHub Actions: `main` → production, `staging` → staging. Pipeline runs `alembic upgrade head` then restarts PM2 processes, followed by a smoke-test job
